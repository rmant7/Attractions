Пользовательские библиотеки (не библиотеки мета) иногда используют методы Selenium. Они также ищут страницу, поле ввода, отправку сообщений и т.д. Например, библиотека instagrapi.
Известные библиотеки либо обращаются к запросам Selenium, либо к свободным функциям Instagram API (Application Programming Interface).
 Единственная проблема, почему скрипты для Instagram не работают или вызывают сбой в работе или блокировку аккаунтов, это то, что методы автоматизации не одобряются компанией Meta.
 А работу Selenium блокировать будет также, потому что это запрещено самой Meta, и они используют свои алгоритмы для поиска ботов и т.д.

Работа с официальным Instagram API требует разрешений.
 Если для Facebook можно включить режим разработчика, то для Instagram нужно пройти полную верификацию, выложить код, написать пользовательское соглашение, подтвердить свою компанию, после чего код проходит тестирование.
 На каждый метод нужно написать детальное объяснение. Все официальные проекты, например Livedune, SMMplanner, Onlypult и другие, имеют (по их словам) разрешение от Meta.
